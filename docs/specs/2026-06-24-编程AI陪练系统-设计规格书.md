﻿# 编程AI陪练系统 - 设计规格书

**日期**：2026-06-24  
**版本**：v1.0  
**技术栈**：Electron + React + Python FastAPI + LangChain + ChromaDB + SQLite

---

## 1. 产品概述

面向全阶段编程学习者的桌面端 AI 陪练系统，覆盖从零基础入门到面试突击的完整学习链路。通过多 Agent 协同架构和 RAG 检索增强，提供评估、教学、练习、陪练、面试一体化体验。

### 1.1 目标用户
- 零基础新手：手把手引导，从 Hello World 起步
- 进阶学习者：已会一门语言，想学新语言或深入框架
- 面试备考者：算法刷题 + 系统设计 + 模拟面试

### 1.2 核心价值
- 本地运行，数据隐私自主可控
- 多 Agent 分工协作，每个环节都有专精 AI 服务
- RAG 知识库驱动的精准内容检索，避免 AI 幻觉
- 自适应难度，根据用户水平动态调整

## 2. 系统架构

### 2.1 技术选型

| 层 | 技术 | 理由 |
|---|------|------|
| 桌面壳 | Electron 28+ | 跨平台桌面，生态成熟，Web 技术栈 |
| 前端 | React 18 + TypeScript + Monaco Editor | 组件化 UI，专业代码编辑器 |
| 后端 | Python 3.11 + FastAPI | AI/ML 生态第一选择，异步高性能 |
| AI 编排 | LangChain + LangGraph | Agent 编排、工具调用、多轮对话管理 |
| 向量数据库 | ChromaDB | 轻量嵌入式，适合本地部署 |
| 关系数据库 | SQLite (via SQLAlchemy) | 零配置，本地持久化 |
| Embedding | bge-large-zh-v1.5 | 中文编程领域表现优秀 |
| 重排序 | bge-reranker-v2-m3 | 提升检索精度 |
| 代码沙箱 | Docker (可选) / subprocess 隔离 | 安全执行用户代码 |
| LLM | 本地模型 + 云端 API 双通道 | 隐私优先，按需切换 |

## 3. 模块详细设计

### 3.1 语言选择模块

**功能**：
- 支持语言：Python, JavaScript, TypeScript, Java, C, C++, Go, Rust, SQL, HTML/CSS
- 按"热度/难度/领域"分类展示
- 支持同时学习多门语言，独立切换学习上下文
- 每门语言展示学习路径预览

**数据模型**：
```python
class Language:
    id: str          # python, javascript, ...
    name: str        # 显示名
    category: str    # 后端/前端/系统/数据
    difficulty: int  # 1-5
    roadmaps: List[Roadmap]  # 关联学习路线
```

### 3.2 基础评估模块 (评估Agent)

**功能**：
- **摸底测验**：15-25 道自适应题目，根据答题情况动态调整难度
- **经验问卷**：3-5 题快速了解编程背景和学习目标
- **能力建模**：输出雷达图（语法基础、数据结构、算法思维、工程能力、领域知识 5 维度）
- **定级**：L1(零基础) ~ L5(可面试)

**评估Agent 职责**：
- 从 RAG 题库中按难度梯度抽取题目
- 分析答题正确率和耗时
- 生成能力画像 JSON，写入用户记忆库

### 3.3 学习计划引擎 (教学Agent + 数据Agent)

**功能**：
- 基于评估结果 + 用户目标（求职/兴趣/学业）+ 可用时间，动态生成学习路径
- 按周拆分阶段目标，每日推荐学习任务
- 支持手动调整和 AI 建议两种模式
- 数据Agent 实时追踪完成率，动态调优计划

**计划结构**：
```
阶段1: 基础语法 (2周)
  ├─ Day 1-3: 变量与数据类型
  ├─ Day 4-6: 控制流
  ├─ Day 7-9: 函数与模块
  └─ Day 10-14: 综合练习 + 阶段测验
阶段2: 核心进阶 (3周)
  ...
```

### 3.4 互动教程模块 (教学Agent)

**功能**：
- **对话式教学**：AI 用自然语言讲解概念，用户可随时追问"为什么""举个例子"
- **代码演示**：每个知识点配可运行代码示例，支持逐行讲解
- **实时运行**：代码块一键运行，即时看到输出
- **课程体系**：预置 Python/JS/Java 等主流语言的标准课程，AI 可在此基础上个性化调整

**教学Agent 职责**：
- 从 RAG 知识库检索教学内容和代码示例
- 根据用户理解程度调整讲解深度
- 支持"换个方式解释""给我更多例子""这里我不懂"等元指令

### 3.5 智能习题系统 (习题Agent)

**功能**：
- **AI 出题**：基于当前学习进度和薄弱点自动生成题目（选择/填空/编程）
- **在线编码**：Monaco Editor 集成，语法高亮 + 智能补全
- **自动批改**：代码运行验证 + 风格检查 + 思路分析
- **分级题库**：L1-L5 难度，支持按知识点/题型筛选
- **思路点评**：不只告诉你对错，还分析你的解法优劣、推荐更优解

**习题Agent 职责**：
- 从 RAG 题库检索类似题目作为参考
- 根据用户水平生成难度适配的新题
- 批改时调用代码沙箱执行，比对预期输出
- 生成包含"正确性/效率/可读性"三维度的批改报告

### 3.6 AI 对话陪练 (陪练Agent)

**功能**：
- **24h 答疑**：贴代码问 bug、问概念、问最佳实践
- **代码审查**：提交一段代码，AI 审查逻辑、风格、安全性
- **Debug 辅助**：描述现象，AI 推断原因并给出修复方案
- **自由对话**：不限编程，可聊学习心态、职业规划

**陪练Agent 职责**：
- 理解用户问题意图
- 从 RAG 知识库检索相关技术文档
- 结合用户历史学习数据给出个性化建议

### 3.7 模拟面试官 (面试Agent)

**功能**：
- **算法面试**：随机出题 → 限时作答 → AI 追问复杂度/边界 → 综合评分
- **系统设计面试**：给定场景（如设计短链接系统）→ 引导式对话 → 评估方案
- **行为面试**：STAR 法则引导，模拟真实面试官追问
- **面试报告**：每场结束输出评分 + 改进建议 + 参考回答

**面试Agent 职责**：
- 模拟真实面试官行为模式（追问、沉默压力、提示时机）
- 从 RAG 题库中按面试难度抽取题目
- 生成量化的面试评分报告

### 3.8 学习数据看板 (数据Agent)

**功能**：
- **进度追踪**：学习时长、完成课程数、做题量、正确率趋势图
- **薄弱点热力图**：按知识点维度可视化弱项
- **能力变化曲线**：定期重评估，追踪各项能力成长
- **学习建议**：基于数据推荐下一步重点方向

### 3.9 成就与激励系统

**功能**：
- 连续打卡天数、累计学习时长里程碑
- 成就徽章（"百题斩""全勤王""面试达人"等）
- 周/月学习报告自动生成

### 3.10 错题本与薄弱点分析

**功能**：
- 自动收录错题，按知识点分类
- 定时推荐错题重做
- 薄弱点定向强化练习自动生成

### 3.11 知识笔记系统

**功能**：
- 学习过程中一键收藏知识点
- AI 自动生成笔记摘要
- 标签分类 + 全文搜索
- 支持导出 Markdown

## 4. Agent 协作流程

### 4.1 编排器 (Orchestrator) 职责

- **意图识别**：解析用户输入，判断属于评估/学习/练习/答疑/面试哪个域
- **任务拆解**：复杂请求拆分为多 Agent 协作子任务
- **Agent 路由**：将子任务分发到对应专精 Agent
- **上下文管理**：维护会话状态，传递 Agent 间共享上下文
- **冲突仲裁**：多 Agent 返回结果不一致时决策

### 4.2 典型场景流程

**场景：新用户首次使用**
```
用户输入 → 编排器 → 评估Agent (摸底测验 + 能力建模)
                  → 教学Agent (生成学习计划)
                  → 数据Agent (初始化用户画像)
```

**场景：学习过程中遇到问题**
```
用户提问 → 编排器 → 陪练Agent (RAG 检索 + 答疑)
                  → 数据Agent (记录薄弱点)
```

**场景：完成一个学习阶段**
```
阶段完成 → 编排器 → 习题Agent (阶段测验)
                  → 数据Agent (更新画像 + 调整计划)
                  → 教学Agent (推荐下一阶段)
```

## 5. RAG 知识库设计

### 5.1 知识库 (Knowledge Base)

| 内容类型 | 来源 | 更新策略 |
|---------|------|---------|
| 编程语言官方文档 | 官网爬取/API | 月度增量更新 |
| 经典教程 | 开源教程/Markdown | 手动导入 |
| 最佳实践/设计模式 | 精选文章 | 季度更新 |
| API 参考 | 官方 SDK 文档 | 跟随版本 |

### 5.2 分级题库 (Question Bank)

| 维度 | 说明 |
|------|------|
| 难度 | L1(入门) ~ L5(面试级) |
| 题型 | 选择、填空、判断、编程、调试 |
| 知识点 | 按语言/框架/领域三级标签 |
| 变体能力 | 一道基础题可自动生成多个变体 |

### 5.3 用户记忆库 (User Memory)

```python
class UserProfile:
    languages: Dict[str, LanguageLevel]  # 各语言能力
    weak_points: List[WeakPoint]         # 薄弱知识点
    mistake_log: List[MistakeRecord]     # 错题记录
    learning_history: List[SessionLog]   # 学习轨迹
    preferences: UserPreferences         # 偏好设置
    goals: List[LearningGoal]            # 学习目标
```

### 5.4 检索策略

- **混合召回**：向量检索 (语义相似) + BM25 (关键词匹配) 加权融合
- **重排序**：bge-reranker-v2-m3 对召回结果精排
- **上下文注入**：检索结果作为 System Prompt 注入对应 Agent
- **缓存策略**：高频查询结果 LRU 缓存，降低延迟

## 6. 数据流

```
用户操作 (Electron 前端)
    │
    ▼
FastAPI WebSocket/HTTP
    │
    ▼
编排器 (Orchestrator)
    │
    ├─→ Agent A ─→ RAG 检索 ─→ LLM 推理 ─→ 返回结果
    ├─→ Agent B ─→ 代码沙箱 ─→ 执行结果
    └─→ 数据Agent ─→ SQLite 读写
    │
    ▼
结果聚合 → 前端渲染
```

## 7. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 桌面框架 | Electron | 成熟生态，React 组件复用，跨平台 |
| 后端语言 | Python | AI/ML 生态，LangChain 原生支持 |
| Agent 框架 | LangGraph | 支持有状态多 Agent 图编排 |
| 向量数据库 | ChromaDB | 轻量嵌入式，零运维 |
| 本地优先 | SQLite + ChromaDB | 数据隐私，离线可用 |
| 代码执行 | Docker + subprocess 双模 | 安全隔离与轻量兼顾 |
| LLM 双通道 | 本地 Ollama + 云端 API | 隐私/能力按需切换 |
